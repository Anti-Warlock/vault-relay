# 第一阶段 Review：结构化文章核心

## 本阶段目的

确认插件是否能够正确理解一篇 Obsidian 长文，而不是直接把 Markdown 当作字符串拆分。

只有结构识别可靠，后续才能稳定生成：

- X Article
- X Thread Cards
- 知乎、公众号等富文本草稿
- 图片、表格、代码块等发布资源

## 使用方法

1. 覆盖 Obsidian 插件目录中的 `main.js`、`manifest.json`、`styles.css`。
2. 重启 Obsidian 并确认插件版本为 `0.4.0`。
3. 打开一篇真实长篇 Markdown。
4. 打开命令面板。
5. 执行 `Obsidian Publish Studio: 检查当前文章结构`。

## 请重点 Review

- 标题数量和层级是否正确。
- 正文段落是否被正确分开。
- 有序列表和无序列表是否被识别。
- 引用和 Obsidian Callout 是否被识别。
- 代码块语言是否正确。
- Markdown 表格是否被识别。
- OSS 远程图片是否显示为远程资源。
- `![[image.png]]` 是否显示为本地资源和发布前警告。
- 是否存在错误或无意义的警告。
- 节点预览顺序是否与原文章一致。

## 本阶段暂不 Review

- X 网页自动填充。
- X 图片上传。
- X Article 样式。
- 图片卡片视觉设计。
- 多平台发布。

这些能力会在结构化文章核心通过 review 后逐步实现。

## 通过标准

用三篇不同结构的真实文章测试：

1. 普通长文。
2. 包含图片、表格和代码的技术文章。
3. 包含 Callout、引用和复杂列表的 Obsidian 笔记。

三篇文章的结构识别基本正确，且没有严重遗漏或顺序错误，即可进入下一阶段：**本地富文本预览与 X Article 输出编译器**。
